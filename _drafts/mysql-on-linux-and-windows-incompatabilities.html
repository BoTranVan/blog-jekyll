While in general MySQL runs the same on both operating systems, here's a short list of some differences between the two.

I wish to avoid stating differences that are obvious, such as "Linux has Unix-Socket and Windows has Named-Pipes", but rather state those differences which imply different functionality or availability of utilities.

  • On Windows, table names are case insensitive. So SELECT * FROM Country is the same as SELECT * FROM country.
  • Moreover, Windows stores table names in lower case. This means replicating from Windows (master) to Linux (slave) may fail due to the fact that the MySQL server on windows regards tables in an altered case than the original, whereas linux expects exact names. This can be solved with setting lower_case_table_names=1 on the linux machine.
  • SET PROFILING and SHOW PROFILE will not work on Windows.
  • MySQL Cluster is not available for Windows at the time of this writing.
  • MySQL Proxy is not available for Windows at the time of this writing.
  • Not all Perl scripts supplied with the MySQL distribution are applicable on Windows. Not all of them are provided on Windows (The MySQL 5.0 windows installation comes with no Perl scripts at all. The 5.1 installation comes with some).
  • Some high availability options are not available on Windows, namely Linux Heartbeat and DRBD. Windows Cluster is only available on Windows.
  • You can expect most of the common 3rd party utilities to run well on Linux: mytop, innotop, mysqlreport, mysqlsla, maatkit etc.
